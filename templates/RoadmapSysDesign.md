# **Roadmap for System Design Interview**

![roadMapSysDesign.png](..%2Fdiagrams%2FroadMapSysDesign.png)

## **Components**
- Functional requirements - features of the system (**Use should be able to...** statements .... )
- Non-Functional Requirements - Qualities of the system (**-lities of the system, availability, reliability, scalability etc)
- Core entities - Data models
- API - Use core entities to satisfy the functional requirements
- High-Level Design - Satisfy the core features
- Deep Dives - Satisfy all non-functional requirements
